Widgets

Fuwafuwa Framework introduces Widgets as a powerful tool to streamline development and enhance UI consistency. These widgets encapsulate common UI elements, such as forms, inputs, and interactive components.

Core Widget Types

Form-based Widgets:

Input Widgets: These widgets represent form input fields and include:

  • ff:input: Basic text input.
  • ff:textarea: Multiline text input.
  • ff:select: Dropdown selection.
  • ff:checkbox: Checkbox input.
  • ff:radio: Radio button group.
  • ff:upload: File upload input.
And other standard HTML input types.

Non-Input Widgets:

  • Card: A container for displaying structured content.
  • Modal: An overlay window for displaying content or forms.
  • Rating: A star-based rating system.
  • Spinner: A loading indicator.
  • Tab: Organize views in tabs.
  • Progress Bar: A loading indicator.

Widgets typically is a tag with ff prefix like <ff:input>, except for button and badge, they are simply html element with class. Widgets for form typically consist of label, input, error text, and explanation text. Common widget attributes are:

  • title: Displays a label or heading for the widget
  • class: Applies additional CSS classes for styling
  • inline: Renders the label and input side by side
  • explanation: Provides additional information about the widget
  • type: Specifies the input type (text, number, date, etc.)
  • errormodel: Links the widget to an error model for displaying validation errors
  • exattr: Defines additional HTML attributes for the explanation element
  • coattr: Defines additional HTML attributes for the widget container
  • required: Indicates whether the input is mandatory.
Other attributes will be assigned to input element.
